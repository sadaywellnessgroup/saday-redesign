Saday Wellness — Master Blueprint

Digital-only mental health consultation ecosystem for India. Single source of truth for product, clinical, technical, and operational decisions. Consolidates two independent competitive scans — SwasthMind CRM (the clinical-CRM reference for Indian mental-health practices) and Tealfeed (the solopreneur booking/coaching platform many Indian MHPs currently default to) — together with the CEO/CTO strategic review, the five-track care architecture, the adaptive intake design, the 30-tool assessment library, and the 6-month execution plan.

1. Executive Summary

Saday Wellness is a digital-only mental health consultation platform targeting Tier 2–3 Indian cities. It is operated by a psychiatrist-CTO founder, a clinical psychologist co-founder, a back-end developer (Claude-assisted), an amateur designer, and 2–3 already-enrolled mental health professionals (MHPs) as launch providers. Budget: ₹1 lakh bootstrapped. Target: working V1 in 6 months.

Product thesis in one sentence

An honest-scope mental health platform that triages every intake into one of five care tracks, uses 30 freely-available validated assessment tools (not just PHQ-9/GAD-7), supports collateral informants where clinically necessary (bipolar, geriatric, pediatric, schizophrenia follow-up), and publishes longitudinal outcome data as its long-term moat.

The four foundational design choices

1. Adaptive intake, not a static form

Language → self/proxy → age → plain-language symptom clusters → adaptive assessments → always-on safety screen → contact. Two visible exits (red-flag hard-redirect and voluntary skip) so the form is never a dead-end.

2. Five care tracks, not one pathway

Wellness · Standard outpatient · Complex (BPD, bipolar, SUD, complex PTSD) · Supported / Adjunct (stable schizophrenia follow-up) · Not-suitable-for-online (acute psychosis / severe withdrawal / anorexia with low BMI / acute suicidality). Each track has its own MHP type, session cadence, pricing, and assessment set.

3. Informant / collateral system

Three tiers of linked accounts (observation only / collateral interview / caregiver support). No other Indian platform has this. Non-negotiable for bipolar, schizophrenia follow-up, dementia, pediatric, and ADHD diagnostic pathways.

4. Outcome data from day one

PHQ-9, GAD-7, mood logs, and population-specific tools captured at fixed treatment milestones (weeks 0/2/4/8/12). Immutable, server-timestamped, episode-of-care linked. This becomes the published evidence base — the real moat.

2A. Tealfeed Scan Synthesis — Booking Polish, Not Clinical Depth

The Tealfeed admin scan (Apr 2026, 45 screens audited, 11 top-nav modules, 14 product/admin areas including Form Builder and Segment Analysis) surfaces a second reference point. Tealfeed is not a clinical platform — it is a creator / coach / consultant booking product that many independent Indian MHPs currently use because nothing better exists for solopreneurs. It tells us what a polished Indian booking product looks like, and equally what it refuses to do.

What Tealfeed does well (Saday should selectively adopt)

  • Bundle product — prepaid multi-session packs (Ongoing / Completed state machine).
  • Coupons engine — code, discount, apply-on, usage limit, start/expiry.
  • Multi-gateway payments — Razorpay + Easebuzz + Tealfeed Pay; Deductions entity (tax / service charge); Manage Payout for team splits.
  • Segment Analysis — filter clients by expert × product × date; export.
  • Form Builder — 12 field types (text, email, number, textarea, dropdown, radio, checkbox, date, time, linear scale, MC grid, checkbox grid). Reusable templates via Form Template Library.
  • Availability engine — timezone, future-booking window, minimum-notice; drag-to-create Day/Week calendar.
  • Global Defaults — service categories (color-coded), session/notification templates, client-setting defaults.
  • Earnings ledger — per-transaction: service, buyer, PG charges, net amount, deductions, coupon, payout status, no-show flag, refund status.
  • Client panel — Follow-up link, Assigned Members, Services Bought, Last Availed, Revenue Earned per client.
  • Org Settings — reschedule.kind / cancellation.kind policy config.
  • My Blogs / Published Articles — built-in content engine for SEO.

What Tealfeed does not do (Saday moat — clinical)

  • No validated assessment instruments — Form Builder is generic; cannot do PHQ-9 scoring, MDQ cut-off logic, or PCL-5 cluster scoring.
  • No symptom-cluster triage or adaptive branching — flat forms only.
  • No care-track model — every buyer is treated the same; no refusal protocol.
  • No safety / crisis escalation — no C-SSRS, no on-call SLA, no red-flag alerts.
  • No SOAP / clinical notes / diagnosis codes / medication list / treatment plan.
  • No informant / collateral workflow — single-buyer identity only.
  • No paediatric / guardian consent architecture.
  • No DPDP / Telemed-specific posture — creator-economy ToS, not health-data compliance.
  • No episode-of-care or longitudinal outcome tracking.
  • Accessibility debt — 102 inputs missing label, 39 screens below WCAG AA contrast, 56 consistency issues (scan-measured).
  • English-only; no Hindi / regional language surface.
  • No prescription surface (letterhead / signature / e-Rx).

Tealfeed patterns Saday explicitly rejects

  • Generic Form Builder as clinical instrument. An MHP can create a "depression screener" in Tealfeed using radio buttons — Saday never allows this. All clinical scoring goes through versioned validated tools in assessment_tool_versions.
  • "Every buyer = client" model. In Tealfeed anyone with a card becomes a client. Saday filters at intake — Track 5 patients are referred out, not onboarded.
  • Drafts / Deleted / Unconfirmed session states visible to buyer. Tealfeed exposes these; Saday keeps clinical workflow state server-side only (HIPAA-style minimum disclosure).
  • Segment export of client list. Tealfeed allows "Export" on Client Panel and Team Members. Saday restricts exports to aggregate / de-identified only; client-level export requires admin + audit-log entry + patient consent.
  • English-only interface. Non-negotiable for Saday: Hindi + IST + INR on day 1.
  • Accessibility debt. Saday launches at WCAG AA contrast / labelled inputs; Tealfeed's 56 consistency issues document what not to ship.
Key insight: Tealfeed is the pricing, booking, and payments reference. SwasthMind is the clinical CRM reference. Saday merges Tealfeed's solopreneur-ergonomics with SwasthMind's clinical surfaces, then adds what neither does: adaptive triage, validated-instrument scoring, informant system, and honest-scope refusal.

4. Five-Track Care Model

Mental health presentations exist on four axes (severity, online-treatability, informant-dependency, population). A single pathway does not serve them. Saday routes every completed intake into one of five tracks. Each track is a distinct care model — different MHP type, session cadence, pricing, and assessment schedule.

Track 1 — Wellness & Subclinical

Relationship difficulties, burnout, grief, adjustment issues, life transitions, work stress, existential concerns. No diagnosable disorder by definition but high volume and commercially valuable.

MHP: Counselling psychologist or trained counsellor (master's level, supervised). Cadence: Fortnightly, 45 min. Price anchor: ₹500–800 / session. Assessments: PSS-10 + PHQ-9 at weeks 0 and 4 (to catch misclassified cases). Re-route risk: BPD, bipolar II, early psychosis all present here first — re-screen at session 3.

Track 2 — Standard Outpatient Clinical

Depression, anxiety, OCD, PTSD, adult ADHD without major comorbidity. Core volume; works well with integrated psychiatry-psychology model.

MHP: Clinical psychologist ± psychiatrist. Cadence: Weekly or fortnightly, 45 min. Price anchor: ₹500 (psychiatric) / ₹800–1200 (therapy). Assessments: PHQ-9 + GAD-7 + tool-specific (OCI-R for OCD, PCL-5 for PTSD, ASRS for ADHD) at weeks 0, 2, 4, 8, 12.

Track 3 — Complex Clinical

BPD, stable bipolar maintenance, moderate eating disorders, SUD in counselling/recovery phase, ADHD + significant comorbidity, complex PTSD. Manageable online but requires structured protocols, higher session frequency, and usually informant involvement.

MHP: DBT-trained psychologist + psychiatrist (split). Cadence: Weekly 60 min + between-session structured check-ins. Price model: Package-based (8-session packages) not per-session. Assessments: Diagnosis-specific + mood log (bipolar) + MSI-BPD retest (BPD) + AUDIT/DAST (SUD) + C-SSRS every visit.

Track 4 — Supported / Adjunct

Stable schizophrenia follow-up, severe bipolar in maintenance, dementia + caregiver. Platform is not primary treating team — a local psychiatrist is. Saday provides monitoring, psychoeducation, caregiver support, medication-adherence structure.

MHP: Psychiatrist (follow-up) + caregiver support counsellor. Cadence: Monthly follow-up + caregiver sessions on demand. Mandatory fields: primary_treating_physician, primary_emergency_contact, local_hospital. Assessments: Clinician-rated (PANSS / BPRS / YMRS / MoCA) during video; informant observation forms weekly during high-risk periods.

Track 5 — Not Suitable for Online Care

Active florid psychosis; acute mania; severe alcohol or benzodiazepine withdrawal (medical emergency); anorexia BMI <16; acute suicidality requiring inpatient observation; active child protection concerns.

Saday response: Intake immediately presents emergency-referral screen with: iCall (9152987821), Vandrevala Foundation (1860-2662-345), nearest NIMHANS centre, state-wise emergency psychiatry contacts. Logged as triage_outcome = not_suitable with reason. No booking offered. This protects the patient and the platform.

Track assignment is automatic at intake, editable by junior counsellor after first conversation. Every patient record stores assigned_track, track_assigned_by, track_changed_log. Track migration (e.g., T2 → T3 after BPD unmasks) is an expected clinical event, not a bug.

5. Adaptive Intake System — The Front Door

The intake is not a form — it is a 7-screen conversational state machine rendering from JSON config. This is the single most differentiated product surface of Saday.

The screen flow

Screen 0 — Language Selection

Rendered before the logo fully loads. Three buttons: English · हिंदी · Other (reveals: Gujarati / Tamil / Telugu / Bengali — coming soon). "Other" does not dead-end — falls back to Hindi/English. Everything from Screen 1 onward renders from a locale JSON (no hardcoded strings anywhere).

Screen 1 — Who Is This For?

"Who are you seeking help for today?" [ For myself ] → Screen 2A [ For someone else — family or friend ] → Screen 2B (proxy branch)

The proxy branch is a separate form tree, not just a different name field. Language shifts to third-person ("she has been forgetting things recently"), consent architecture changes, and age-band routing becomes primary.

Screen 2A — Self: Basic Context

Age band: [ Under 18 ] [ 18–25 ] [ 26–40 ] [ 41–60 ] [ 60+ ] Gender: [ Male ] [ Female ] [ Non-binary / Other ] [ Prefer not to say ]

Age band (not exact age) drives tool selection. Under 18 → paediatric pathway (V1: hard redirect to "we'll call you"; V1.5: full adolescent pathway). 60+ → GDS-15 substitutes for PHQ-9; AD8 enters the routing table.

Screen 2B — Proxy: Context Collection

Relationship: [ Parent of a child ] [ Spouse / Partner ] [ Adult child of an elderly parent ] [ Friend ] [ Other family member ] Approximate age: [ <12 ] [ 12–17 ] [ 18–40 ] [ 41–60 ] [ 60+ ] Are they aware you're seeking help on their behalf? [ Yes ] [ No ] [ I'm not sure ]

"Aware" = No → non-blocking note: "We'll discuss the best way to involve them when you speak with our team." This is informed consent territory; proxy alone cannot authorize treatment for a competent adult.

Screen 3 — Symptom Cluster Selection (the heart of the form)

13 plain-language clusters + "something else" + "I'm not sure." Multi-select. Each has a brief elaboration on tap. Floating "Skip — book directly" button visible here and on every subsequent screen.

[ ] Feeling very sad, empty, or losing interest in things [ ] Worry, anxiety, or fear that's hard to control [ ] Extreme mood swings — very high and very low periods [ ] Trouble focusing, restlessness, acting without thinking [ ] Drinking, substance use, or feeling unable to stop [ ] Something traumatic that keeps coming back to you [ ] Thoughts or urges that repeat and feel distressing [ ] Seeing, hearing, or believing things others say aren't real ← red-flag [ ] Memory problems, confusion, or unclear thinking [ ] Difficult relationship with food or your body [ ] Intense emotional pain in relationships, or not knowing who you are [ ] Complete exhaustion, burnout, or feeling overwhelmed [ ] Sleep problems that are affecting your life [ ] Something else I haven't seen here [ ] I'm not sure — I just know something is wrong ────────────────────── [ Skip — I want to book directly ]

Exit rules at Screen 3

0 selected + "I'm not sure": soft route → "That's okay. Our team will figure it out together with you. Book a first session."
1–2 selected: proceed to adaptive assessments (Screens 4+).
3+ selected: soft redirect → "You've mentioned several areas. Rather than continue with forms, we'd recommend speaking directly with one of our professionals." [ Book a Session ] [ Continue with form anyway ]
Red-flag cluster selected (psychosis; substance withdrawal symptoms; active suicidality on later screen): hard redirect to crisis resources + 2-hour contact commitment. No "continue" option.

Screens 4+ — Adaptive Assessments

Tools triggered by the Screen 3 selections and Screen 2 age band. One question per mobile screen. Progress indicator ("Question 3 of 9"). Back button. Auto-save every answer to localStorage.

Cap: 25 questions total across all selected tools. If exceeded, serve gate versions (PHQ-2, GAD-2, AUDIT-C) and note full_assessment_deferred_to_session = true in the backend submission.

Transitions are human sentences, not clinical headings: "You mentioned feeling low or losing interest. These 9 questions will help us understand how you've been feeling."

Screen 5 — Safety Screen (always, for everyone, never skipped)

"In the past two weeks, have you had any thoughts of hurting yourself or ending your life?" [ No ] [ Passing thoughts only ] [ Thinking about it seriously ] [ Rather not say ] "Is there anything you need urgently right now?" [ No, I'm okay ] [ I'm in a difficult situation right now ]

"Thinking about it seriously" OR "Rather not say" combined with any other indicator → safety protocol fires before form completion. Crisis numbers shown. Option to be contacted within 1 hour. safety_flag written to submission as an immutable field.

Screen 6 — Contact & Completion

Name (first name is sufficient) Phone (WhatsApp preferred) Best time to reach you Preferred MHP type: [ ] Recommend for me [ ] Psychiatrist [ ] Psychologist / therapist [ ] Initial chat with a counsellor [ Submit and See Available Appointments ]
Architecture rule: The form is a pure state machine. States = language · who_for · basic_context · clusters · assessment_sequence[] · safety · contact. Every transition is a config-driven rule. Every exit rule is a config value. Changing a threshold is a database update, not a deployment.

8. Informant / Collateral System — A True Novel Feature

No Indian mental-health platform currently has this. It is non-negotiable for bipolar, schizophrenia follow-up, dementia, paediatric, and ADHD diagnostic pathways. Design must respect Indian family dynamics (family involvement is culturally normal) without breaking therapeutic confidentiality (family involvement is sometimes exactly what patient does not want).

Three-tier linked account model

TierWhoWhat they can doWhat they can't see
L1 — Observation onlyCaregiver invited by adult patientSubmit weekly / monthly observation form. Sees their own submissions.No session notes. No assessment results. No appointment history.
L2 — Collateral interviewInformant invited for a specific sessionJoin one scheduled collateral interview session (video). Documented separately as "collateral interview", not regular session.Not recurring — each session requires fresh invite.
L3 — Caregiver / guardianParent of child patient; adult child of dementia patientManages patient account, books appointments, joins sessions. Has own billable caregiver-support appointments.—

Consent and access rules

  • L1 invitation requires patient to set access level at time of invite, with explicit "You can change or revoke this anytime" notice.
  • L1/L2 access is scoped by linked_account_id + episode_of_care_id — revoked if patient ends the episode.
  • Patient can revoke linked-account access without explanation. Revocation is logged but not explained to the linked party.
  • For L3 on paediatric <12 patients, the guardian IS the account-holder; there is no separate "patient account".
  • For L3 on adolescent 12–17 patients, both accounts exist; disclosure rules follow NMC Telemedicine Guidelines 2020 confidentiality carve-outs (suicidality / abuse / harm-to-others break confidentiality to guardian).
  • If MHP believes L1 collateral is clinically essential (bipolar, schizophrenia) but patient refuses: document refusal in notes, adjust plan accordingly, do not force.

9. Special Populations

Adult ADHD

ASRS-v1.1 at intake → if positive, book 90-min extended assessment appointment with DIVA-2.0-capable psychologist. Informant (parent / partner) corroboration for childhood symptoms is the gold standard — build L2 collateral interview into the pathway. Always rule out bipolar II, anxiety, sleep disorder as alternatives before diagnosis. Medication decisions live in V2 with proper prescription entity.

Borderline Personality Disorder (the stealth presentation)

Rarely self-identified. Junior counsellor needs training to spot red flags (history of multiple therapist changes, "difficult" label, intense fear of abandonment, self-harm without suicidal intent, rapid idealization-devaluation cycles). BPD cannot live in Track 2 — requires DBT-trained psychologist, structured between-session coaching windows (not ad-hoc WhatsApp), documented safety plan. If you don't have DBT-trained MHP, refer honestly.

Bipolar disorder

Mood log feature is the highest-value clinical tool of the entire platform. Two fields, 30 seconds, daily WhatsApp prompt: "How are you today? (1 sad ↔ 10 very high) | Sleep last night: ___ hours." MHP sees 3-month trajectory at a glance before each session. Informant (spouse / parent) weekly observation form during high-risk periods — hypomania is universally insight-free. Lithium / valproate level monitoring: recurring task reminder + lab result upload + MHP flag when overdue. V2: lab API integration (Thyrocare, Dr. Lal PathLabs).

Substance Use Disorders

NDPS Act creates legal sensitivity — platform takes harm-reduction clinical approach, addiction is a medical condition, non-judgmental access. Intake must assess substance, frequency, quantity (AUDIT/DAST), last use, previous detox history, and current withdrawal symptoms. Active alcohol / benzodiazepine withdrawal = medical emergency = Track 5 exit. Platform offers: MI in early change stages, relapse prevention, family psychoeducation, co-occurring management. Does NOT offer: medically-supervised detox or residential rehab.

Schizophrenia & psychosis (follow-up only — Track 4)

Stable, treatment-adherent, insight-intact patients only. Acute = Track 5. Critical feature: family/caregiver portal (L1) with monthly observation (sleep changes, social withdrawal, medication adherence, unusual behaviour) that flags to MHP on concerning patterns. Assessment tools (PANSS, BPRS) are clinician-administered during video session. Mandatory primary_treating_physician + local_hospital fields.

Geriatric (60+)

Substantially different across every dimension. PHQ-9 over-detects somatic aging symptoms — use GDS-15. Cognitive screening (MoCA, clinician-administered) standard. Polypharmacy + psychotropics = anticholinergic / fall risk monitoring. Depression often presents as memory / pain / withdrawal, not "sad". Family-assisted onboarding pathway is first-class: son/daughter sets up account, manages appointments, is present in sessions. MHP profile tag: "comfortable_with_geriatric" required for routing. Caregiver burden is frequently the more acute presenting problem — caregiver-support sessions are billable.

Paediatric & adolescent

Consent architecture by age:
<12: Parent / legal guardian is account holder. Child is identified patient but has no platform access. MHP may meet child alone for rapport but treatment contract is with parents.
12–17: Adolescent self-referral allowed with simplified consent + explicit confidentiality exceptions (suicidality, abuse, harm-to-others). Parental consent required for treatment to begin UNLESS adolescent is seeking help related to parental abuse (child protection protocol overrides).
Assessment tools change: CDI-2 (depression), SCARED (anxiety), Vanderbilt/SNAP-IV (ADHD), SDQ (behavioural), M-CHAT (autism screen <4y), CRAFFT (substance). Teacher-rated forms require one-time-access-link system (V2). Autism diagnostic assessment is out of scope — platform does post-diagnosis support only. Adolescent eating disorders require medical threshold for in-person referral.

10. Safety & Crisis Protocol

This is the single most legally and ethically consequential section of the platform. Every item below is P0 and must be live before the first patient books.

Always-on safety detection

  • Screen 5 (suicidality screen) fires for every intake, every time. Not conditional on clusters. Not skippable.
  • PHQ-9 Q9 ≥ 1 auto-triggers C-SSRS screener on next question.
  • C-SSRS ideation ≥ 4 OR any behaviour item fires immediate crisis screen + on-call clinician WhatsApp alert.
  • Red-flag clusters (psychosis, active withdrawal symptoms) hard-redirect at Screen 3 with no "continue" option.
  • Post-intake submission with any safety flag enters clinician_review_queue with SLA <2 hours (not 4). Junior counsellor cannot clear. Senior only.

12. CTO Non-Negotiables — Architecture Decisions You Cannot Easily Undo

1. Multi-tenant-ready from day 1

Single tenant at launch but every table has organization_id. RLS policies scope by it. V2 white-label = config change, not rebuild.

2. Role-based access in the first commit

Five roles: patient · mhp · admin · counsellor · linked_account. Supabase RLS policies tested with a cross-patient access attempt before launch.

3. Immutable clinical data

Assessments, notes (post-sign), safety plans — append-only. Edits create new versions. Server timestamps only, never client.

4. Audit log from commit #1

Not a V2 feature. Append-only table. Every mutation to PHI writes a row. DPDP Section 8(5) requires demonstrable processing records.

5. Indian data residency

Mumbai region only. Supabase ap-south-1. No US regions even for backups. Vendor DPAs (Razorpay, MSG91, Daily.co) confirmed India-compliant.

6. HttpOnly cookies for session tokens

Not sessionStorage (Swasthmind's SEC-04 mistake). Supabase Auth default. CSRF token on state-changing requests.

7. Field-level encryption for identifiers

Phone, Aadhaar (if ever collected — avoid), bank details (MHP payouts). Use pgcrypto or app-layer envelope encryption.

8. No PHI in logs — ever

Logger redacts name, message body, clinical content. Only user IDs and action types. Every new log statement reviewed.

9. Webhook retry + manual reconciliation

Razorpay webhook handler with retry + dead-letter queue. Daily reconciliation screen: Razorpay settlements vs. appointments. Fixes Swasthmind G-08 gap and webhook-miss failure mode.

10. Hard rate limits on outbound messaging

MSG91 per-patient cap (e.g., 3 WhatsApp messages / day). Prevents runaway-loop bills. Set in config, enforced at send time.

11. Automated daily backups

Supabase point-in-time recovery + nightly pg_dump to encrypted S3 (Mumbai). Quarterly restore drill.

12. Error monitoring on day 1

Sentry free tier. 20 minutes to set up. Tells you when something breaks before patients do.

Enforcing cross-cutting consistency despite Claude-assisted development

The Claude-generated code failure mode: each module is correct in isolation but auth handling, error shapes, and transaction boundaries diverge across modules. The fix is an architecture document the developer writes in Week 1 that defines: auth middleware, error response shape, DB transaction rules, PHI redaction rule, naming conventions. Every Claude prompt references it. Weekly code review looks for deviations, not details.
