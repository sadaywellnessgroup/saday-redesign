Scaling Roadmap

One risk-ascending path from a thin, verifiable core → the ₹1L product → the full OG blueprint. Plus a marketing-site phase up front to beat the Odoo clock.

The rule

Build only what you can personally verify is safe. Prove it with real patients. Then let agents expand the surface — fastest where it's lowest-risk.

Staged risk-ascending, not feature-descending. Each stage has a hard exit gate — you don't start the next until it passes. That's what keeps a solo build from quietly shipping an unsafe PHI surface.

Founder-leverage map

Where your hands help — and where they don't

Your scarce resource isn't typing (agents cover that) — it's review time on security-critical code and wall-clock on calendar items. Stage the build so you never hold more unverified surface than you can review at once.

🟢

You own it

Frontend, component system, intake UX, accessibility, EN/HI i18n, design system, patient-app polish, and clinical-logic correctness (you're the psychiatrist).

Lead. This is your edge. · Agent leverage: high

🟡

Agents draft · you verify

RLS policies, auth, payments/webhooks, audit log, SOAP record integrity, API routes, migrations. Read every line. Buy one external security review at the P1 and P3 gates.

Reviewer-in-chief. · Verification is the bottleneck

🔴

Calendar, not code

Meta/MSG91 template approval, Razorpay live KYC, pen-test, clinical sign-off, DPDP/medico-legal review. Start early, run in parallel.

Agent leverage: none

M

Marketing Site & Content Migration

~1–2 wks · do first

Goal: a content-complete, on-brand static marketing site live on sadaywellness.com before the Odoo subscription lapses. No accounts, no DB, no PHI — brochure-ware. Zero security surface, ships independent of every app decision.

Ships

  • • Home — hero, "Why?" stat, services, stats band, testimonial (done in mockup)
  • • About / Team — 8 real bios
  • • Services — 8 services + 2 programs
  • • Contact — phone/email/address + no-backend form
  • • Privacy + Terms — written fresh

Watch-outs

  • ⚠️ Live Support Groups card is Odoo lorem-ipsum — rewrite
  • ⚠️ /terms placeholder & /privacy 404 — health brand needs real legal pages
  • ⚠️ Shop / courses / memberships die with Odoo (eCommerce module) — decision needed
  • ↳ Export Odoo data (PDFs, contacts) before lapse
Inputs already in hand: the full sadaywellness.com content audit (verbatim) + mockups/saday-home.html (modernized homepage already built). Booking CTA keeps pointing to existing external booking (Tealfeed/Swasthmind) during the gap. Deploy to Cloudflare Pages, point DNS, done.
Exit gate M → P0: all pages live on the real domain · real Privacy + Terms published · contact form delivering · booking CTA valid · Odoo data exported · subscription safely cancellable.
P0

Foundation

~1 wk

Goal: every downstream decision is locked and the infra exists. No product yet.

Lock first (baked into the schema)

  1. Safety screen — ASQ vs C-SSRS (D2)
  2. Proxy/age — 18+ self-only for P1
  3. Admin vs notes — B6 RLS-deny?
  4. Audit — append-only + revoke UPDATE/DELETE
  5. PHI encryption — disk+RLS now, Vault later
  6. Brand — purple vs sage (gates all frontend)

Build

  • • Supabase in ap-south-1, RLS-default-deny on
  • • Next.js 14 + TS + Tailwind scaffold + CI
  • • Auth skeleton + current_user_id() bridge
  • • Design tokens + base component library
Exit gate P0 → P1: repo runs · logged-in user resolves through current_user_id() · a non-owner SELECT returns zero rows (default-deny proven).
P1

Thin Safe Core

~3–5 wks · the verification floor

Goal: the smallest app a real patient can use safely. Pilot with a few consenting patients. This is where you spend most of your review budget.

Ships (full loop)

  • • Sign-up + hard 18+ gate; doctor/admin TOTP
  • • Short self-only adaptive intake
  • • Safety screen → crisis resources + flag (no auto-book)
  • • PHQ-9 + GAD-7, versioned, immutable
  • • Book → Razorpay → confirm
  • • Jitsi video (lobby + password)
  • • SOAP sign-&-lock + B6 admin-deny
  • • Signed-PDF prescription · append-only audit · DSR

Not in P1

Tracks/routing · proxy · care-plan PDF · mood logs · any tool beyond PHQ-9/GAD-7 · ledger · outcomes · messaging · materials · Vault crypto · Merkle audit.

🟢 You own ~60%

Patient app, intake UX, SOAP editor, i18n, design system in anger.

🟡 Verify line-by-line

RLS, TOTP, Razorpay webhook, audit grants, immutability + sign-lock, B6 deny.

Exit gate P1 → P2 (the big one): RLS attack-test suite passes (cross-patient blocked, admin can't read note bodies, signed note immutable, audit rows can't be edited) · a real Razorpay payment reconciles · you clinically review the safety path · one external security review · 3–5 pilot patients complete the full loop with no safety/privacy defect.
If this gate doesn't pass, nothing else matters.
P2

Product MVP ≈ ₹1L scope

~3–4 wks on top

Goal: turn the safe core into a sellable product by adding the ₹1L differentiators — now on a foundation you've proven safe.

Adds

  • • Care-plan PDF (the differentiator)
  • • Proxy — adult-for-adult only (minors still excluded) + consent flow
  • • Richer intake (more clusters; no full track engine)
  • • Cancellation/refund surfaced (manual via Razorpay)
  • • Patient portal: sessions, receipts, history
  • • Consent ledger (granular, versioned)
  • • Light admin: patient list, schedule, flag queue

The one re-expansion of liability

Proxy returns — gate it with its own consent + access-control review.

Still not in P2

5/7-track engine · full 12-tool library · ledger/payouts · outcomes · informant tiers · Vault crypto · Merkle audit.

Exit gate P2 → P3: proxy access-control attack-tested (can't exceed scope, can't act for a minor) · paying patients running · refund/cancellation working · DPDP posture reviewed · real demand signal.
P3

Full OG Blueprint · the moat

~3–4 mo on top

Goal: the V1.5/V2 system — 34-table model, data moat, multi-clinician ops. Built surface-by-surface; each addition is now low-risk on a proven base, and these are the parts agents do best.

1. Full 12-tool library — highest leverage, lowest risk
2. Care-track engine — 5 vs 7 decided first
3. Outcomes dashboard — the moat
4. Earnings ledger + payouts — GST/TDS
5. 3-tier informant system — reopens minor question
6. Hardened compliance — Vault crypto + Merkle audit
7. Ops surface — KYC, materials, messaging, calendar
8. Daily.co video with signed URLs
Exit gate (launch-grade): pen-test passed · clinical sign-off · DPDP/medico-legal review · MSG91/Meta templates live · backup/restore verified · Merkle anchor verifiable.
