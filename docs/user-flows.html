User Flow Specification

End-to-end journeys for every actor on the Saday Wellness platform

Phase 0 → Phase 2 5 Roles Action Plan v1.0
🆕

New User — Patient

First visit · no existing account · never used the platform

1
Landing / Discovery
User arrives via organic search, referral link, or WhatsApp share card. Sees hero + value prop.
Screen: Home / Landing CTA: "Start Healing" or "Talk to Someone"
2
Language Selection
User picks English or Hindi. Persisted to localStorage and used throughout intake.
Screen 0: Welcome EN / HI toggle
3
Who Is This For?
User states whether they are seeking help for themselves, their child, or someone else (informant linkage triggered later).
Screen 1: Who-Is-This-For
Branch
Myself → Continue intake as primary patient
My Child / Family → Flag for informant / guardian consent later (Screen 6)
Someone Else → Redirect: suggest they share the link with that person
4
Age Band
Determines eligibility and appropriate track. Under-18 requires guardian consent.
Screen 2: Age Band
Branch
18–60 → Standard adult intake continues
Under 18 → Guardian consent screen injected before Screen 6
60+ → Flag for geriatric considerations; Track 2 preferred MHPs surfaced
5
Symptom Clusters
Adaptive multi-select of concern areas (sleep, mood, anxiety, relationships, substance, etc.). Cluster routing logic runs in real-time. Red-flag cluster triggers Track 5 short-circuit immediately.
Screen 3: Symptom Clusters Adaptive — hides irrelevant clusters
🔴 Safety Short-Circuit If user selects "thoughts of self-harm / suicide" cluster → skip to C-SSRS probe (Screen 5b) immediately. Positive C-SSRS → Crisis Resources screen with Vandrevala + iCall. Do NOT proceed to booking.
C-SSRS positive → Track 5 · Crisis
6
Duration & Impact
How long, how much it's affecting work / relationships. Refines track severity.
Screen 4: Duration / Impact
7
Contextual Questions
Short adaptive follow-ups based on clusters selected (e.g., PHQ-2 bridging items for mood; AUDIT-C for substance). EN + HI culturally adapted phrasing.
Screen 5: Contextual Questions Tools: PHQ-2, AUDIT-C, ISI snippets
8
Consent + Summary
Displays recommended track + brief "what happens next." Collects: Privacy Policy consent, Telemed consent, (if under-18: Guardian consent). User creates account at this point — email + OTP only.
Screen 6: Consent + Summary Account creation: email + OTP
Assigned Track
Track 1 Mild / Stress / Wellbeing — Counsellor
Track 2 Moderate / Mood / Anxiety — Psychologist
Track 3 Complex / Trauma / OCD — Senior Psychologist
Track 4 Psychiatric / Medication — Psychiatrist
Track 5 Crisis — Junior Counsellor escalation → Psychiatrist
9
Book First Session
Platform surfaces MHPs matching the assigned track. User picks MHP, sees bio + languages. Picks date/time slot from availability calendar.
Screen: Pick MHP Screen: Pick Slot Filter by track, language, availability
10
Payment
Review booking summary. Pay via Razorpay (UPI, card, netbanking). Option to buy session package (4-pack / 8-pack at discount).
Screen: Review + Pay Razorpay checkout Package option shown
Payment Outcome
Success → Booking confirmed; WhatsApp + email confirmation sent
Failure → Retry screen; slot held for 10 min
11
Booking Confirmed → Patient Portal
Lands on dashboard. Sees upcoming session card. Receives 24h WhatsApp reminder automatically. Pre-session check-in prompt appears 1h before session.
Screen: Patient Dashboard WhatsApp: booking-confirmation template WhatsApp: reminder-24h template
12
Video Session
Join via Daily.co room link from portal. Session timer runs. MHP can mark complete / no-show / reschedule from their side.
Screen: Session Room Jitsi fallback if Daily.co unavailable
13
Post-Session: Summary + Follow-up
Patient sees MHP-shared session summary (next steps, homework, medication list if applicable). 24h post-session follow-up check-in WhatsApp sent automatically.
Screen: Session Summary WhatsApp: post-session-check-in template Materials: psychoeducation PDFs available
End of new-user flow → merges with Returning User journey from this point
🔄

Returning User — Patient

Existing account · completed at least one session · logging back in

1
Login
Email + OTP (no password). HttpOnly cookie session. "Remember this device" sets 30-day cookie.
Screen: Login / OTP OTP via email
2
Patient Dashboard
Lands on dashboard showing: upcoming session(s), package balance, recent session summaries, safety plan (if set), materials library shortcut.
Screen: Dashboard Package balance visible if purchased
3
Returning User — Key Actions
From dashboard the patient has multiple paths available simultaneously.
Available Actions
Book Next → Book follow-up with same or new MHP; package consumed if active
Reschedule → Within policy window: free. Outside window: fee per org policy
Cancel → Within policy: refund to source. Outside: credit or platform policy
View Summary → Read past session summaries shared by MHP
Safety Plan → View / edit their personal safety plan
Materials → Browse psychoeducation library (EN + HI PDFs)
Buy Package → Purchase 4-session or 8-session pack
4
Pre-Session Check-in (1h before)
1-tap mood rating prompt appears in portal + WhatsApp. Optional free-text "anything you want your MHP to know." Submitted to MHP dashboard before session.
Screen: Pre-Session Check-In WhatsApp prompt → links to portal
5
Session → Follow-up Loop
Video session → MHP writes SOAP → session summary shared → 24h follow-up check-in. Patient responds to check-in. Repeats for subsequent sessions in care plan.
Screen: Session Room Screen: Session Summary WhatsApp: post-session-check-in (24h)
6
Ongoing: Informant Invitation (optional)
Patient can invite a family member / caregiver as an informant (L1/L2/L3). Informant receives consent request. If accepted, can submit structured observations visible to MHP only.
Screen: Informant Invite Consent-gated — patient controls access
Returning patient loop is continuous until discharge or self-exit
🧠

Psychiatrist (MHP — Track 4 Lead)

Medical doctor · prescribes medications · handles complex psychiatric cases · requires TOTP 2FA

1
Onboarding (One-time)
Admin registers psychiatrist. Psychiatrist receives invite email → sets up TOTP 2FA (mandatory). Submits: NMC registration number, specialties, languages, availability windows, per-session fee.
Screen: MHP Onboarding Form TOTP 2FA required before any patient access NMC reg. verified by admin
2
Login
Email + password + TOTP 2FA code. Session is HttpOnly cookie. Auto-logout after 8h inactivity.
Screen: MHP Login 2FA mandatory — no bypass
3
MHP Dashboard — Daily View
Lands on dashboard with: today's session schedule, client cards (last-session / next-session / care-track / safety-flag), incident queue (C-SSRS alerts), unread follow-up responses.
Screen: MHP Dashboard Safety flags highlighted in red Week-view toggle available
4
Pre-Session Prep
Opens patient's client card. Reviews: intake summary, assigned track, past SOAP notes, medication list, assessment scores, patient's pre-session check-in note. Sets session intent.
Screen: Client Card Assessment history: PHQ-9, GAD-7, MDQ trends Pre-session check-in response shown
5
Video Session
Joins Daily.co room from dashboard. Session is 1:1. Timer shown. Can mark: Completed / No-show / Rescheduled.
Screen: Session (MHP view) Jitsi fallback available
6
SOAP Note + Clinical Documentation
After session: writes Subjective, Objective, Assessment, Plan. Adds/updates medication list (not e-Rx — list only). Assigns ICD-11 diagnosis code to episode of care. Note is versioned and immutable once signed.
Screen: SOAP Editor Medication list (not e-Rx) ICD-11 code lookup Immutable after sign — cannot edit
7
Session Summary — Share with Patient
Selects what to share: next steps, homework, key points (not full SOAP). Patient can see only this curated summary. Triggers follow-up check-in WhatsApp in 24h.
Screen: Session Summary Editor Patient notified via WhatsApp
8
Assessment Administration (Between Sessions)
Can assign formal tools to patient: PHQ-9, MDQ, AUDIT-C, PCL-5, etc. Patient receives WhatsApp link → completes in portal. Scores appear in client card automatically.
Screen: Assign Assessment 12 V1 tools available Scores auto-computed + trended
9
Safety Escalation Handling
If C-SSRS alert arrives (from patient intake or in-session disclosure): receives MSG91 alert within 2h SLA. Reviews patient. Activates safety plan review. Logs escalation outcome in incident queue.
Screen: Incident Queue 2h SLA — timer shown Safety plan review + update
🔴 P0 — Never skip Every C-SSRS-positive case must have a logged response. Admin sees SLA adherence. JC covers out-of-hours; psychiatrist on-call as backup.
10
Earnings + Payout View
Sees personal earnings ledger: session fee, platform commission deduction, net, payout status. Cannot edit — read-only view of finance.
Screen: Earnings Read-only — payouts by admin
Repeating loop: Dashboard → Prep → Session → SOAP → Summary → Follow-up reads
💬

Psychologist (MHP — Tracks 1–3)

Non-prescribing clinician · therapy-focused · handles Tracks 1, 2, 3 patients

Key difference from Psychiatrist: Psychologist flow is 90% identical to Psychiatrist. The differences are: (1) no medication list access, (2) no ICD-11 diagnosis entry (assessment summary only), (3) assigned Tracks 1–3 only, (4) no Track 4/5 patients visible in roster. All safety escalation responsibilities remain identical.
1
Onboarding (One-time)
Admin registers. Psychologist receives invite → sets TOTP 2FA. Submits: RCI registration number, specialties (CBT, DBT, ACT etc.), languages, availability, fee.
Screen: MHP Onboarding Form RCI reg. verified by admin TOTP 2FA mandatory
2
Dashboard + Roster
Same structure as psychiatrist dashboard but roster filtered to Tracks 1–3 clients only. Safety flag column present — Track 3 clients (complex) may have elevated risk.
Screen: MHP Dashboard T1 T2 T3
3
Session + SOAP Notes
Same flow as psychiatrist: prep → video → SOAP (S/O/A/P). Assessment Plan section includes therapeutic modality and homework. No medication section shown in SOAP editor for psychologist role.
Screen: SOAP Editor (Psychologist view) No medication list section Therapeutic homework field
4
Track Escalation Recommendation
If psychologist determines a Track 1–3 client needs psychiatric evaluation (medication, complex diagnosis), they flag for escalation. Admin / psychiatrist sees flag. Patient is offered a transfer booking with psychiatrist.
Screen: Client Card → Escalate Triggers admin notification Patient consent required for track change
5
Informant Observations — View
If patient has linked an informant, psychologist can view informant observations (L1 access). Useful for paediatric / adolescent cases, bipolar monitoring.
Screen: Client Card → Informant Tab Only visible with patient consent on file
Same repeating loop as psychiatrist. Discharge triggers episode-of-care closure.
⚙️

Main Admin

Platform operator · full read access · no patient PHI in raw form · manages MHPs, policies, finance, incidents

PHI Access Note: Admin sees aggregate data, booking records, payment records, and incident queue — but does NOT see clinical notes (SOAP), assessment scores, or raw symptom responses. RLS enforces this.
1
Login
Email + password + TOTP 2FA. TOTP mandatory. Separate admin subdomain or route (/admin). Session auto-expires after 4h.
Screen: Admin Login TOTP mandatory · 4h session
2
Admin Dashboard — Overview
Headline KPIs: total active patients, sessions this week, revenue MTD, no-show rate, open incidents, SLA adherence %. Live metric cards refreshed periodically.
Screen: Admin Dashboard Bookings · Revenue · Utilisation · Safety KPIs
3
MHP Management
Add / edit / deactivate MHPs. Verify NMC/RCI registration. Set per-MHP commission rate. View MHP utilisation (sessions, no-show rate, earnings, payout status). Trigger manual payout.
Screen: MHP Roster Screen: MHP Profile + Earnings Trigger payout Deactivate / re-activate
4
Bookings Management
Full booking list with filters (date, MHP, track, status). Can view booking details: patient name, MHP, slot, payment status. Cannot see clinical notes.
Screen: Bookings List Filters: MHP · Track · Status · Date
5
Revenue + Payments
Earnings ledger view: gross, PG charges, platform commission, net to MHP, refund status, payout status. Razorpay webhook DLQ visible — dead-letter items flagged for manual reconciliation.
Screen: Revenue Ledger Screen: Webhook DLQ / Reconciliation Initiate refund (within policy window) DLQ alert if webhook failures > 0
6
Incident Queue — Safety
All C-SSRS-positive events appear here with SLA timer. Status: Open / Acknowledged / Resolved. Admin tracks MHP response time. If SLA breached: escalation alert fired to on-call lead.
Screen: Incident Queue SLA: 2h from alert to MHP acknowledgement Mark resolved · add admin note
🔴 P0 Admin must never close an incident without a logged MHP response. Audit log captures every state change.
7
Organization Policies
Configure platform-wide rules: cancellation policy (hours window, refund type), reschedule policy, minimum booking notice, session package prices, commission rates, intake cluster config.
Screen: Org Settings Changes audit-logged · cannot roll back silently
8
Compliance + Legal Artifact Management
View versioned legal documents (privacy policy, telemed consent, MHP service agreement, adolescent consent). Each doc has a version stamp and the date it was accepted by each user. Upload new versions; old versions archived.
Screen: Legal Documents Upload new version Consent acceptance log per user
9
Audit Log Viewer
Append-only log of every PHI write, role escalation attempt, and admin action. Filterable by user, table, action, date range. Export to CSV for medico-legal review.
Screen: Audit Log Read-only · append-only · tamper-evident Export CSV
Admin has no session-end — access is role-based and persistent while employed

Care Track Reference

Intake routing output — determines MHP type and tools assigned

Track 1 Mild / Stress / Wellbeing

MHP: Junior Counsellor
Tools: PSS-10, ISI
Typical presenting: Work stress, sleep issues, mild anxiety, adjustment
Session cadence: 4-session pack typical

Track 2 Moderate / Mood / Anxiety

MHP: Psychologist (RCI)
Tools: PHQ-9, GAD-7, SCOFF
Typical presenting: Depression, anxiety disorders, eating concerns
Session cadence: 8-session pack typical

Track 3 Complex / Trauma / OCD

MHP: Senior Psychologist
Tools: PCL-5, OCI-R, MSI-BPD
Typical presenting: PTSD, OCD, borderline features, complex trauma
Session cadence: Ongoing; 8-session packs

Track 4 Psychiatric / Medication

MHP: Psychiatrist (NMC)
Tools: MDQ, ASRS-v1.1, AUDIT-C
Typical presenting: Bipolar, ADHD, psychosis, substance, medication mgmt
Session cadence: Variable; follow-up 4-6 weeks

Track 5 Crisis

MHP: JC first-line → Psychiatrist on-call
Tools: C-SSRS (trigger)
Presenting: Active SI/HI, acute psychosis, immediate risk
Action: Crisis resources shown; no booking offered; SLA 2h alert
